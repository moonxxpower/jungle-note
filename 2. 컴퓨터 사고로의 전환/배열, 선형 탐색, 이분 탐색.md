1️⃣ **배열**

**배열** (Array) ****은 컴퓨터 과학에서 기초적인 자료 구조 중 하나로써 묶음 단위로 값을 저장한다.

**배열의 크기**는 배열에 데이터 원소가 얼마나 들어 있는지 알려주고, **배열의 인덱스**는 특정 데이터가 배열의 어디에 있는지 알려 주는 숫자이다.

(*) **자료구조**는 논리적인 관계로 이루어진 데이터 구성을 말한다. 즉, **자료구조**는 데이터가 모여 있는 구조로써 컴퓨터에서 처리해야 하는 많은 데이터를 모아 효율적으로 관리하고 구조화하기 위해 알아야 한다.

- 파이썬에서는 배열을 리스트 (List) 와 튜플 (Tuple) 형태로 구현할 수 있다. [데이터 컨테이너]

(1) 파이썬에서 배열 원소의 자료형은 서로 다른 자료형을 같이 저장할 수 있으며, 배열 원소 자체를 배열에 저장할 수도 있다.

(2) **리스트**는 원소를 변경할 수 있는 뮤터블 (mutable), **튜플**은 ****원소를 변경할 수 없는 ****이뮤터블 ****(immutable) 자료형이다.

(*) **뮤터블**, **이뮤터블**

변수에 어떤 값을 대입하면, 값이 아니라 식별 번호가 바뀐다. `“python2”` 라는 문자열 객체는 아무도 자신을 참조하지 않기 때문에 가비지 컬렉터에 의해 자동으로 메모리에서 소멸된다.

파이썬 기본 데이터 타입 중 정수, 실수, 문자열, 튜플이 대표적인 immutable 객체이고, 리스트, 딕셔너리가 mutable 객체이다.

- **자료 구조 연산**

배열 같은 자료 구조의 성능을 알려면 코드가 자료 구조와 일반적으로 어떻게 상호작용 하는지 분석해야 한다.

대부분의 자료 구조는 네 가지 기본 방법을 사용하며, 이를 연산이라 부른다.

(1) **읽기**

읽기는 자료 구조 내 특정 위치를 찾아보는 것이다.

컴퓨터가 배열의 특정 인덱스에 있는 값을 읽을 때, 컴퓨터는 모든 메모리 주소에 한 번에 갈 수 있고, 배열을 할당할 때 어떤 메모리 주소에서 시작하는지도 기록해 두기 때문에 한 번의 단계로 바로 갈 수 있다.

(2) **검색**

검색은 자료 구조 내에서 특정 값을 찾는 것이다.

컴퓨터는 모든 메모리 주소에 한 번에 접근할 수 있지만, 각 메모리 주소에 어떤 값이 있는지 바로 알지 못하기 때문에 각 셀을 한 번에 하나씩 조사해야 한다. 이러한 과정을 **선형 탐색** (Linear Search) 이라 부른다.

<img width="564" alt="3" src="https://github.com/moonxxpower/jungle-note/assets/118599217/9ef6fe19-0685-45c2-a7cd-cdc2cbe26929">

컴퓨터에서 배열은 꼭 다음처럼 보인다!

(*) **선형 탐색**은 직선 모양 (선형) 으로 늘어선 배열에서 검색하는 경우에 원하는 키값을 가진 원소를 찾을 때까지 맨 앞부터 스캔하여 순서대로 검색하는 알고리즘이다.

```python
# while
def seq_search(a, key):
	i = 0
	
	while True:
	
		# 검색에 실패하여 -1을 반환
		if i == len(a):
			return -1          
			
		# 검색에 성공하여 현재 검사한 배열의 인덱스를 반환 
		if a[i] == key:
			return i  
			   
		i += 1
```

(3) **삽입**

삽입은 자료 구조에 새로운 값을 추가하는 것이다.

배열에 새 데이터를 삽입하는 연산은 배열의 어디에 데이터를 삽입하는가에 따라 효율성이 다르다. 컴퓨터는 배열을 할당할 때, 항상 배열의 크기를 기록하기 때문에 **배열 끝**에 삽입할 경우, 한 단계면 된다.

하지만, 배열의 **맨 처음**이나 **중간**에 데이터를 삽입할 경우, 삽입할 공간을 만들기 위해 많은 데이터 조각을 이동시켜야 하므로 단계가 늘어난다. 

특히 데이터를 배열의 **맨 앞**에 삽입할 때, 배열 내 모든 값을 한 셀씩 오른쪽으로 옮겨야 하기 때문에 최악의 시나리오이다.

(4) **삭제**

삭제는 자료 구조에 새로운 값을 추가하는 것이다.

실제적으로 데이터를 삭제하고, 빈 공간을 메꾸기 위해 (빈 공간이 있으면 효율적이지 않다.) 데이터 이동이 필요하다.

배열의 **첫 번째 원소**를 삭제하는 것이 삽입과 마찬가지로 최악의 시나리오이다. 이렇게 되면 인덱스 0이 비게 되고, 남아 있는 모든 원소를 왼쪽으로 이동시켜 빈 공간을 채워야 되기 때문이다.

2️⃣ **정렬된 배열**

**정렬된 배열** (Ordered Array) 는 값이 항상 순서대로 있어야 한다. 

- **자료 구조 연산**

(1) **삽입**

값을 추가할 때마다 적절한 셀에 넣어 배열의 값을 정렬된 상태로 유지한다.

따라서 정렬된 배열에 삽입할 때는 항상 실제 삽입 전에 검색을 먼저 수행해서 삽입할 올바른 위치를 정해야 함을 알 수 있다.

흥미롭게도 삽입에 필요한 단계 수는 새 값이 정렬된 배열 어디에 놓이게 되든 비슷하다. 값이 정렬된 배열 앞 부분에 놓이면, 비교가 줄어들고 이동이 늘어난다. 값이 뒷 부분에 놓이면 비교가 늘어나고, 이동이 줄어든다. 새 값이 배열 맨 끝에 놓이면 이동하지 않아도 되니 단계 수가 가장 적다.

(2) **검색**

**선형 탐색 관점**에서 **전형적인 배열**의 경우, `[17, 3, 75, 202, 80]`이라는 전형적인 배열이 있다고 했을 때, (배열에 없는) 22라는 값을 찾으려면 22가 배열 어디든 있을 수 있으므로 모든 원소를 하나도 빠짐없이 검색해야 한다. 배열의 끝에 도달하기 전에 검색을 멈추는 경우는 원하는 값을 찾았을 때뿐이다.

하지만, **정렬된 배열**에서는 값이 배열에 들어 있지 않을 때, 검색을 더 빨리 멈출 수 있다. 정렬된 배열 `[3, 17, 75, 80, 202]`에서 22를 찾는다고 할 때, 75에 도달하면 더 이상 22가 오른쪽에 있을 수 없으므로 바로 검색을 중단할 수 있다.

선형 검색은 특정 상황에서 전형적인 배열보다 정렬된 배열에서 단계 수가 더 적게 걸린다. 하지만, 찾으려는 값이 배열의 마지막 값이거나 마지막 값보다 크면 마찬가지로 모든 셀을 검색해야 끝난다. 언뜻 보면 일반적인 배열과 정렬된 배열이 효율성 면에서 대단한 차이가 없어 보이고, 적어도 최악의 시나리오에서는 그렇다. 

정렬된 배열이 전형적인 배열보다 크게 두드러진 장점은 다른 검색 알고리즘을 쓸 수 있다는 점이다. 이러한 알고리즘을 **이분 탐색** (Binary Search) 이라 부르며, 이분 탐색은 선형 탐색보다 훨씬 빠르다.

3️⃣ **이분 탐색**

**이분 탐색**은 정렬된 배열에만 쓸 수 있다.

전형적인 배열은 값의 순서가 뒤죽박죽이어서 주어진 값의 왼쪽에서 찾을지, 오른쪽에서 찾을지 절대 알 수 없다.

이분 탐색을 수행할 수 있다는 것이 정렬된 배열의 장점 중 하나이다.

- 정렬된 배열에서 값 7을 찾는다고 할 때, 이분 탐색은 다음과 같이 동작한다.

(1) 가운데 셀부터 검색을 시작한다. 

<img width="579" alt="4" src="https://github.com/moonxxpower/jungle-note/assets/118599217/67747a5c-361b-4bb4-9edd-94b780ecf5ff">

값이 9로 들어 났으므로 7은 왼쪽 어딘가에 있다고 판단할 수 있다. 배열에 있는 셀의 절반, 즉 9보다 오른쪽에 있는 모든 셀과 9 자신을 제거한다.

<img width="573" alt="5" src="https://github.com/moonxxpower/jungle-note/assets/118599217/7ffef3fd-e9a6-429a-a991-d6914ccb9d55">

(2) 9보다 왼쪽에 있는 셀들 중 가운데 값을 확인한다. 가운데 값이 두 개이므로 임의로 왼쪽 값을 선택한다.

<img width="575" alt="6" src="https://github.com/moonxxpower/jungle-note/assets/118599217/e9bd3a66-3049-499b-85f3-f55a6e0fee67">

셀의 값이 4이기 때문에 7은 오른쪽 어딘가에 있을 것이다. 따라서 4와 그 왼쪽 셀을 제거한다.

<img width="572" alt="7" src="https://github.com/moonxxpower/jungle-note/assets/118599217/a1300b7d-b98f-4b1b-aae5-8f62eb8bb8ff">

(3) 7일 수 있는 남은 셀 두 개 중 임의로 왼쪽 셀을 선택한다.

<img width="566" alt="8" src="https://github.com/moonxxpower/jungle-note/assets/118599217/b4ee6532-e960-47a8-85e6-61151f056cf6">

(4) 마지막 남은 셀을 확인한다. 만약 여기에 7이 없다면, 이 정렬된 배열에는 7이 없다는 뜻이다.

<img width="573" alt="9" src="https://github.com/moonxxpower/jungle-note/assets/118599217/46c29d3c-fbab-4efe-813e-1db677d6b6a8">

```python
def bin_search(a, key):

	# 검색 범위 맨 앞 원소의 인덱스
	pl = 0    
	
	# 검색 범위 맨 끝 원소의 인덱스 
	pr = len(a)-1
	
	while True:
	
		# 중앙 원소의 인덱스
		pc = (pl+pr) // 2
		
		# 검색 성공
		if a[pc] == key:
			return pc
			
		# 검색 범위를 뒤쪽 절반으로 좁힘
		elif a[pc] < key:
			pl = pc+1
			
		# 검색 범위를 앞쪽 절반으로 좁힙
		else:
			pr = pc-1
		if pl > pr:
			break
		
		# 검색 실패
		return -1
```

4️⃣ **이분 탐색과 선형 탐색**

<img width="489" alt="10" src="https://github.com/moonxxpower/jungle-note/assets/118599217/81723c0d-f597-4b77-b97d-4531e28ef10d">

**선형 탐색**에서는 원소 수만큼 단계가 필요하다. 따라서 배열의 원소 수를 두 배로 늘릴 때마다 검색에 필요한 단계 수도 두 배로 늘어난다. 

**이분 탐색**에서는 배열의 원소 수를 두 배로 늘릴 때마다 한 단계만 늘어난다.

정렬된 배열이 모든 상황에서 빠른 것은 아니다. 정렬된 배열의 삽입은 일반 배열보다 느리다. 하지만, 정렬된 배열을 사용하면, 삽입은 다소 느리지만, 검색은 훨씬 빠르다. 사용에 따라 장단점을 따져 사용하는 것이 필요하다.
